# Как протестировать приложение сейчас

## A) Быстрый тест веб-версии (локально)

### 1) Запустите локальный сервер
```bash
python3 -m http.server 8080
```
Откройте: `http://localhost:8080`

### 2) Проверки в браузере
- Вкладка **Калькулятор**: введите данные, проверьте пересчёт давления/литров/баков.
- Вкладка **Погода**: нажмите "Обновить погоду", убедитесь что карточки 7 дней появились.
- Вкладка **Склад/Обработки**: проверьте фильтры и сортировку.

### 3) Проверка PWA
- DevTools → **Application** → Manifest: должен быть валиден.
- DevTools → **Service Workers**: `sw.js` зарегистрирован и активен.
- Установите приложение через кнопку "Установить на Android" (если браузер показывает prompt).

---

## B) Тест Android-версии (через Android Studio)

### 1) Подготовка Android-проекта
```bash
npm run android:bootstrap
```

### 2) Открыть Android Studio
```bash
npm run cap:open
```

### 3) Запустить на эмуляторе/телефоне
- Нажмите **Run** (зелёная кнопка).
- Выберите эмулятор или подключённый телефон (USB debugging).

### 4) Проверить офлайн-режим
- Запустите приложение, загрузите главную страницу.
- Отключите интернет на устройстве.
- Перезапустите приложение и убедитесь, что базовый интерфейс открывается из кэша.

---

## C) Сборка APK для ручной проверки
```bash
npm run android:build
```
APK: `android/app/build/outputs/apk/debug/app-debug.apk`

---

## D) Что делать, если не запускается
- Ошибка `npm install`: проверьте доступ к `registry.npmjs.org`.
- Ошибка Gradle/SDK: в Android Studio откройте **SDK Manager** и установите недостающие SDK/Build Tools.
- После изменения web-файлов всегда запускайте:
```bash
npm run cap:sync
```
