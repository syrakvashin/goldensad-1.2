# Как улучшить приложение «ГолденСад»

## 1) Надёжность и качество
- Добавить базовые автоматические проверки (например, ESLint + Prettier), чтобы снижать риск регрессий в большом `script.js`.
- Вынести API-адреса в конфиг и предусмотреть fallback/timeout для сетевых запросов.
- Добавить пользовательские уведомления о сетевых ошибках с повторной попыткой.

## 2) Архитектура фронтенда
- Разделить `script.js` на модули по доменам: `calc`, `weather`, `stock`, `treatments`, `blocks`.
- Выделить слой `state` и `storage`, чтобы уменьшить связность и упростить сопровождение.
- Перейти на сборку (Vite/Webpack) для удобной структуры, минификации и проверки типов (JSDoc/TypeScript).

## 3) UX и интерфейс
- Добавить валидацию числовых полей (границы, подсказки, единицы измерения).
- Показывать skeleton/loading состояния отдельно для каждого виджета.
- Добавить экспорт таблиц (CSV/Excel) для «Склада» и «Обработок».

## 4) Производительность
- Кэшировать результаты API (погода/склад/обработки) с TTL в `localStorage`.
- Исключить лишние перерисовки DOM при фильтрации (fragment + батч-обновления).

## 5) Безопасность и эксплуатация
- Добавить CSP, `referrer-policy`, корректные `rel` у внешних ссылок (частично уже есть).
- Централизовать обработку ошибок и логирование.
- Настроить CI: линт, базовые e2e-проверки, сборка на каждый commit.
